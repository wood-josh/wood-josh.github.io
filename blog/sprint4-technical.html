Pseudocode

Is a coding specific problem-solving technique that really helps lay everything that you need to do before you get into the nitty gritty coding side of it all. This is done using comments in plain English to explain what your code is going to need to do before writing the functioning code that you need. Because it is plain English guidelines of what you are going to need to do and not code is doesn’t need to be super in depth and functioning, Pseudocode acts as a great way to get onto paper what you need to do and get an idea of the projects flow without having to get into that projects underlying details.

The rubber ducky method

Or the longer more technical term the rubber duck debugging method is another very useful way to problem solve while coding. The term rubber ducking became popular from a book about a programmer that carried around a duck all the time and had to explain each line of his code to the duck as if it could understand plain English. Now you don’t actually need a rubber duck to do this, any inanimate object will do but for best results use another person. Explaining what your code does out loud is going to help you understand what’s happening and in turn what’s going wrong in your code, if you have someone else that you are explaining it to then you are going to understand what is going on even better.

Reading error messages and Console.loggin

Although not the same things they do coincide a little bit as console.logging helps keep track of where you are up to, what is assigned to what and what you code is doing along with really helping in showing what are causing the error messages to show up. Error messages are sometime a little bit like a guide on what is going wrong or at least help you locate where it is going wrong.

Googling

The name kind of just says it all but this tends to the problem-solving method I rely on the most and have the most success with. There a million of people that have gone down this path before me and a lot of those people really enjoy making guides and videos that help make that journey easier for other people. There is so much information out there and if you are facing a problem in your code, likelihood is that someone else has experienced that same issue before, made a post on a forum about it and then 50 people have replied to his answer with different solutions to the issue with very detailed descriptions about what there solutions do.
